# 共享小说平台 - 社区投票与治理机制设计文档

## 一、治理机制概述

### 1.1 治理理念
共享小说平台采用**分层治理**模式，形成"股份投票权 + 荣誉投票权 + 社区公投"三级治理体系，确保：
- 资本与创意的平衡
- 长期利益与短期诉求的协调
- 专业决策与民主参与的结合

### 1.2 治理层级

```
┌─────────────────────────────────────────────────────────┐
│                    社区治理体系                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │  股东大会   │    │  荣誉议会   │    │  全体公投   │ │
│  │（核心决策） │    │（规则制定） │    │（重大事项） │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│         │                   │                   │       │
│         ▼                   ▼                   ▼       │
│  ┌─────────────────────────────────────────────────┐   │
│  │              社区理事会（执行机构）               │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 二、投票权体系

### 2.1 投票权类型

#### A. 股份投票权（核心决策权）
**获取方式**：持有平台股份的创作者自动获得

| 股份数量 | 投票权重 | 备注 |
|---------|---------|------|
| 0.01% - 0.1% | 1票/0.01%股份 | 基础权重 |
| 0.1% - 1% | 1.2票/0.01%股份 | 1.2倍加成 |
| 1% - 5% | 1.5票/0.01%股份 | 1.5倍加成 |
| > 5% | 2票/0.01%股份 | 2倍加成（封顶） |

**投票范围**：
- 平台盈利分配方案
- 理事会成员选举
- 重大战略方向
- 修改治理规则

#### B. 荣誉投票权（规则制定权）
**获取方式**：通过社区贡献累积

| 贡献类型 | 换算比例 | 备注 |
|---------|---------|------|
| 累计投月票 | 1000张月票 = 1票 | 永久有效 |
| 优质内容创作 | 作品被接纳 = 10票 | 一次性 |
| 社区服务 | 纠错/评论获赞100次 = 1票 | 可累积 |
| 邀请新创作者 | 邀请1人达标 = 5票 | 一次性 |

**投票范围**：
- 社区规则制定与修改
- 内容审核标准调整
- 月票阈值建议
- 新功能提案

#### C. 普通投票权（意见表达权）
**获取方式**：所有实名注册用户

| 用户状态 | 投票权重 |
|---------|---------|
| 普通读者 | 1票 |
| 活跃读者（LV3+） | 2票 |
| 付费用户 | 3票 |

**投票范围**：
- 作品质量反馈
- 功能需求调研
- 活动主题选择
- 非约束性建议

### 2.2 投票权重上限

为防止过度集中，设置权重上限：

| 投票权类型 | 单人/单机构上限 |
|-----------|--------------|
| 股份投票权 | 总票数的20% |
| 荣誉投票权 | 500票 |
| 普通投票权 | 10票 |

## 三、投票场景矩阵

### 3.1 按事项类型分类

#### 类别一：利益分配类（股东大会专属）

| 事项 | 触发条件 | 通过门槛 | 投票权类型 |
|------|---------|---------|-----------|
| 利润分配方案 | 季度/年度 | 股份权≥67%同意 | 仅股份投票权 |
| 平台费用调整 | 理事会提案 | 股份权≥51%同意 | 仅股份投票权 |
| 股份增发/回购 | 理事会提案 | 股份权≥75%同意 | 仅股份投票权 |
| 投资项目决策 | >100万元 | 股份权≥67%同意 | 仅股份投票权 |

#### 类别二：规则制定类（荣誉议会专属）

| 事项 | 触发条件 | 通过门槛 | 投票权类型 |
|------|---------|---------|-----------|
| 新规则提案 | 10人联署 | 荣誉权≥51%同意 | 仅荣誉投票权 |
| 规则修改 | 20人联署 | 荣誉权≥60%同意 | 仅荣誉投票权 |
| 审核标准调整 | 理事会提案 | 荣誉权≥51%同意 | 仅荣誉投票权 |
| 违规处罚标准 | 理事会提案 | 荣誉权≥67%同意 | 仅荣誉投票权 |

#### 类别三：战略方向类（双重投票）

| 事项 | 触发条件 | 通过门槛 | 投票权类型 |
|------|---------|---------|-----------|
| 主线宇宙扩充 | 月票达标申请 | 双重≥51%同意 | 股份权+荣誉权 |
| 平台品牌定位 | 理事会提案 | 双重≥51%同意 | 股份权+荣誉权 |
| 重大技术升级 | >50万元 | 双重≥60%同意 | 股份权+荣誉权 |
| 国际化战略 | 理事会提案 | 双重≥67%同意 | 股份权+荣誉权 |

**双重投票机制**：需股份投票权 AND 荣誉投票权分别通过

#### 类别四：社区事项类（全体公投）

| 事项 | 触发条件 | 通过门槛 | 投票权类型 |
|------|---------|---------|-----------|
| 理事会罢免 | 30%股东联署 | 全体≥67%同意 | 所有人1人1票 |
| 核心规则修改 | 20%社区联署 | 全体≥60%同意 | 所有人1人1票 |
| 平台解散/合并 | 理事会提案 | 全体≥75%同意 | 所有人1人1票 |
| 紧急争议裁决 | 申诉提交 | 全体≥51%同意 | 所有人1人1票 |

#### 类别五：内容决策类（混合投票）

| 事项 | 触发条件 | 通过门槛 | 投票权类型 |
|------|---------|---------|-----------|
| 作品接纳审核 | 月票达标申请 | 审核小组+月票加权 | 专业+民主 |
| 角色命运投票 | 作者发起 | 月票持有者 | 月票权重 |
| 平行线升主线 | 连续3个月达标 | 自动+否决机制 | 自动触发 |
| 内容争议裁决 | 用户申诉 | 陪审团投票 | 随机抽选 |

### 3.2 按紧急程度分类

#### 常规决策
- 时间周期：每月固定时间（如每月1-5日投票）
- 议题范围：规则修改、利润分配、常规决策
- 提前公示：至少7天

#### 紧急决策
- 触发条件：重大漏洞、法律风险、资金安全
- 时间周期：24-48小时内完成
- 议题范围：紧急止损、系统修复、危机应对
- 加速机制：简化流程、缩短公示期

#### 临时决策
- 触发条件：特殊事件、外部机会
- 时间周期：3-7天
- 议题范围：合作提案、市场机会
- 灵活机制：可随时发起

## 四、投票流程设计

### 4.1 标准投票流程

```
┌──────────────┐
│  1. 议案发起  │ ← 任何符合条件的用户/机构可发起
└──────┬───────┘
       ▼
┌──────────────┐
│  2. 联署征集  │ ← 达到最低联署数（10-50人，视事项级别）
└──────┬───────┘
       ▼
┌──────────────┐
│  3. 理事会审核 │ ← 形式审查（是否合规、是否重复）
└──────┬───────┘
       ▼
┌──────────────┐
│  4. 公示期    │ ← 7天公示，收集反馈和补充方案
└──────┬───────┘
       ▼
┌──────────────┐
│  5. 辩论期    │ ← 正反方陈述，社区讨论
└──────┬───────┘
       ▼
┌──────────────┐
│  6. 投票期    │ ← 3-7天投票时间
└──────┬───────┘
       ▼
┌──────────────┐
│  7. 结果公示  │ ← 公开投票详情和执行时间
└──────┬───────┘
       ▼
┌──────────────┐
│  8. 执行/复议 │ ← 执行决策或进入争议解决流程
└──────────────┘
```

### 4.2 快速通道流程（紧急事项）

```
议案发起 → 理事会紧急评估（2小时）→ 加密公示（4小时）
→ 加速投票（24小时）→ 结果执行
```

### 4.3 投票期规则

| 事项类型 | 公示期 | 辩论期 | 投票期 | 总周期 |
|---------|-------|-------|-------|-------|
| 利益分配 | 7天 | 3天 | 5天 | 15天 |
| 规则制定 | 7天 | 5天 | 7天 | 19天 |
| 重大战略 | 14天 | 7天 | 7天 | 28天 |
| 紧急事项 | 4小时 | 跳过 | 24小时 | 28小时 |
| 临时决策 | 3天 | 2天 | 3天 | 8天 |

## 五、投票执行机制

### 5.1 投票方式

#### 线上投票系统（主要方式）
- 匿名投票：公开议题，匿名选择
- 实时统计：投票进度实时公开
- 修改机制：投票期内可修改选择
- 到期提醒：到期前24小时推送通知

#### 投票确认机制
- 双因素认证：密码+短信/邮箱验证
- 投票回执：投票后生成唯一回执码
- 区块链存证：重要投票上链，不可篡改

### 5.2 弃权处理

| 投票类型 | 弃权处理方式 |
|---------|-------------|
| 股东大会 | 弃权不计入分母，按实际投票计算比例 |
| 荣誉议会 | 弃权视为"不反对"，不影响通过率 |
| 全体公投 | 弃权视为放弃权利，不计入统计 |
| 内容决策 | 弃权视为中立，按有效票计算 |

### 5.3 委托投票

#### 适用场景
- 股东长期无法参与
- 荣誉议员临时缺席

#### 委托规则
- 需书面/电子授权
- 单次委托有效期最长6个月
- 可随时撤销委托
- 受托人投票权重叠加

#### 限制
- 单人最多接受5份委托
- 不能委托给理事会成员
- 委托关系公开透明

## 六、争议解决机制

### 6.1 争议类型分级

#### 一级争议（轻微）
**定义**：投票流程瑕疵、信息不充分、理解歧义

**处理流程**：
1. 当事人提出异议（24小时内）
2. 理事会调解（48小时）
3. 补充信息或重新投票（如需）

**案例**：
- 公示期信息不全
- 投票系统故障
- 表述有歧义

#### 二级争议（中等）
**定义**：投票结果争议、程序违规、利益冲突

**处理流程**：
1. 提交书面申诉（3天内）
2. 社区陪审团随机抽选（7人）
3. 陪审团听证调查（7天）
4. 裁决结果公示（有约束力）

**案例**：
- 怀疑刷票/贿选
- 理事会利益冲突
- 投票权重计算错误

#### 三级争议（重大）
**定义**：系统性问题、根本性冲突、罢免/弹劾

**处理流程**：
1. 30%以上联署发起
2. 成立独立调查委员会（5人专家）
3. 全体公投表决
4. 执行裁决（需≥67%同意）

**案例**：
- 理事会成员严重违规
- 治理机制根本性缺陷
- 平台存续危机

### 6.2 社区陪审团机制

#### 陪审员资格
- 注册时长≥6个月
- 无违规记录
- 非议案相关方
- 随机抽选+回避制度

#### 陪审团职责
- 听取双方陈述
- 查阅相关证据
- 独立裁决争议
- 出具裁决报告（公开）

#### 陪审团规模
| 争议级别 | 陪审团人数 | 裁决规则 |
|---------|-----------|---------|
| 一级 | 3人 | 简单多数 |
| 二级 | 7人 | 2/3多数 |
| 三级 | 15人 | 3/4多数 |

### 6.3 上诉机制

#### 上诉条件
- 有新证据
- 程序严重违规
- 裁决明显不公

#### 上诉流程
1. 裁决后3天内提起上诉
2. 提交上诉理由和证据
3. 更高级别机构复核
4. 终审裁决（不可上诉）

#### 上诉路径
```
社区陪审团 → 理事会复核 → 荣誉议会裁决 → 全体公投终审
```

### 6.4 争议解决时效

| 争议级别 | 提交期限 | 处理期限 | 总时长 |
|---------|---------|---------|-------|
| 一级 | 24小时 | 3天 | ≤4天 |
| 二级 | 3天 | 10天 | ≤13天 |
| 三级 | 7天 | 30天 | ≤37天 |

## 七、理事会机制

### 7.1 理事会组成

#### 成员构成（共9人）
| 来源 | 人数 | 选举方式 | 任期 |
|-----|------|---------|------|
| 创作者代表 | 3人 | 股东大会选举 | 2年 |
| 读者代表 | 2人 | 荣誉议会选举 | 2年 |
| 独立专家 | 2人 | 理事会提名+股东大会批准 | 2年 |
| 平台运营 | 2人 | 理事会互选 | 1年 |

#### 理事会职责
- 执行股东大会和荣誉议会决议
- 管理平台日常运营
- 提交议案和年度报告
- 处理紧急事项
- 组织投票流程

### 7.2 理事会选举

#### 候选人资格
- 注册时长≥1年
- 无重大违规记录
- 获10人以上联署提名
- 承诺履职时间≥10小时/周

#### 选举流程
1. 提名期（14天）
2. 竞选期（7天，候选人陈述）
3. 投票期（7天）
4. 结果公示
5. 就职

#### 选举规则
- 采用**单次可转移投票（STV）**制
- 保证少数群体代表性
- 得票最高者当选

### 7.3 理事会罢免

#### 罢免触发条件
1. 严重违反社区规则
2. 连续3次不履行职责
3. 30%以上有投票权人联署
4. 刑事犯罪

#### 罢免流程
1. 提交罢免议案
2. 成立独立调查委员会
3. 举行全体公投
4. ≥67%同意即罢免
5. 补选新理事

## 八、投票激励机制

### 8.1 参与激励

#### 投票参与奖励
| 参与度 | 奖励 |
|-------|------|
| 每参与1次投票 | +5经验值 |
| 连续3个月全勤 | +20张月票 |
| 提交优质议案（通过） | +100张月票 |
| 议案联署（通过） | +10张月票 |

#### 社区贡献积分
- 投票：1次=1分
- 提案：1次=10分
- 辩论发言：1次=2分
- 陪审履职：1次=20分

积分用途：
- 兑换月票（100分=10张）
- 兑换特殊徽章
- 优先体验新功能

### 8.2 优质提案激励

#### 提案资助计划
- 理事会评估优质提案
- 给予最高5000元开发资助
- 成功落地后奖励1000张月票

#### 创新奖励
- 年度最佳提案：1000张月票+永久荣誉
- 年度最佳辩手：500张月票
- 年度最佳陪审员：300张月票

## 九、防操纵机制

### 9.1 投票安全

#### 技术防护
- 双因素认证强制开启
- 设备指纹识别
- 异常登录检测
- 区块链投票存证

#### 行为分析
- 投票时间规律分析
- IP地址聚类检测
- 账号关联性分析
- 机器学习识别异常模式

### 9.2 利益冲突管理

#### 申报制度
- 理事会成员需申报利益冲突
- 陪审团成员需回避相关方
- 严重隐瞒资格取消

#### 回避规则
- 与议案有直接利益关系者不得投票
- 亲属关系需回避
- 经济往来≥1000元需回避

### 9.3 贿选打击

#### 贿选定义
- 用月票/现金换取投票
- 承诺特殊利益换取支持
- 组织有偿投票

#### 打击措施
- 一经查实：永久封禁账号
- 追缴非法所得
- 实名黑名单
- 法律追究（如构成犯罪）

#### 举报奖励
- 举报贿选：500张月票
- 提供证据：额外300张月票

## 十、特殊投票机制

### 10.1 内容决策投票

#### 作品接纳审核
**审核小组**（7人）：
- 创作者代表3人（随机抽选）
- 读者代表3人（随机抽选）
- 专家代表1人（固定）

**审核流程**：
1. 月票达标自动触发
2. 审核小组评审（7天）
3. 月票加权投票（15天）
4. ≥51%同意即接纳
5. 创作者获相应股份

#### 月票加权投票
| 月票数 | 投票权重 |
|-------|---------|
| 1-10张 | 1票 |
| 11-50张 | 3票 |
| 51-200张 | 10票 |
| 201-1000张 | 50票 |
| >1000张 | 200票 |

### 10.2 角色命运投票

#### 发起条件
- 作者主动发起
- 月票≥1000的作品
- 关键剧情节点

#### 投票规则
- 仅给该作品投过月票的读者可参与
- 投票权重=投给该作品的月票数
- 24小时快速投票
- 简单多数决定

#### 限制
- 每季度最多1次
- 不能决定主角生死
- 不能推翻核心设定

### 10.3 平行线升级投票

#### 自动升级机制
- 支线/平行线作品连续3个月达到上级阈值
- 自动触发升级审核
- 无需投票，仅程序性审核

#### 降级保护机制
- 主线作品连续6个月不达标才降级
- 降级前1个月缓冲期

## 十一、投票数据公开

### 11.1 公开原则

#### 完全公开
- 议案全文
- 投票结果（总数、比例）
- 理事会会议纪要
- 陪审团裁决报告

#### 部分公开
- 各方得票数（不公开个人投票）
- 理事会表决记录（公开谁支持谁反对）
- 财务报告（脱敏处理）

#### 保密
- 个人投票选择（匿名投票）
- 未通过的敏感提案
- 个人隐私信息
- 商业机密

### 11.2 数据存档

#### 存档期限
- 所有投票记录：永久保存
- 会议记录：5年
- 财务数据：10年
- 争议裁决：永久

#### 存档方式
- 区块链存证（重要投票）
- 分布式存储（常规数据）
- 定期备份（多地容灾）

## 十三、风险与应对

### 13.1 主要风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 投票率过低 | 决策代表性不足 | 激励机制+默认选项+强制披露 |
| 资本垄断 | 小股东被边缘化 | 投票权重上限+荣誉权制衡 |
| 恶意否决 | 决策瘫痪 | 提高否决门槛+紧急通道 |
| 投票疲劳 | 参与度下降 | 合并投票+委托机制 |
| 群体极化 | 社区分裂 | 辩论机制+陪审团裁决 |

### 13.2 监控指标
- 投票参与率（目标≥40%）
- 投票争议率（目标≤5%）
- 决策执行率（目标≥90%）
- 社区满意度（季度调查）
- 理事会支持率（年度评议）

## 十四、总结

本设计的核心原则：

1. **分层治理，各司其职**
   - 股东大会：利益分配
   - 荣誉议会：规则制定
   - 全体公投：重大事项

2. **制衡机制，防止垄断**
   - 投票权重上限
   - 双重投票机制
   - 利益冲突回避

3. **争议解决，有章可循**
   - 三级争议分类
   - 社区陪审团
   - 上诉终审机制

4. **激励参与，活力社区**
   - 多元激励体系
   - 荣誉系统
   - 数据公开透明
