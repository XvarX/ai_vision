## 一、核心概念

### 平台定位
类似GitHub的小说创作平台，作者创作主线内容，读者可以在任意节点创建分支，创作平行宇宙故事。通过社区投票机制对内容进行分级，根据不同级别进行收益分成。

### 核心原则

| 原则 | 说明 |
|------|------|
| **三位一体** | 股份 = 投票权 = 收益权，权责对等 |
| **创作者优先** | 作品被接纳+月票达标→自动获股，确保新血液注入 |
| **保护少数** | 多重机制保护小股东权益，防止"多数人暴政" |
| **激励相容** | 所有治理机制激励成员为社区创造价值 |
| **渐进放权** | 随社区成熟，逐步降低主作者特权，向去中心化过渡 |
| **包容性多元** | 创作冲突优先用"平行宇宙法"解决，而非简单否定 |

## 二、层级体系

### 2.1 内容分级

| 层级 | 定义 | 创作者权限 | 投票权 | 收益权 |
|------|------|-----------|--------|--------|
| **主线** | 主作者原创内容，决定世界观发展 | 主作者 | 核心股1.5x | 1.5x |
| **支线** | 能融入主线宇宙的内容 | 自动获股 | 贡献股1.0x | 1.0x |
| **平行线** | 不能融入主线宇宙但被认可 | 自动获股 | 贡献股1.0x | 1.0x |
| **混沌线** | 未获社区接纳，读者自嗨 | 无 | 无 | 无 |

### 2.2 层级判定标准

**支线（能融入主线宇宙）：**
- 不与主线设定冲突
- 补充主线内容（配角视角、番外等）
- 主线可以引用/提及
- 社区投票通过

**平行线（不能融入主线宇宙但有价值）：**
- 与主线设定有冲突但质量优秀
- 改变主线剧情走向
- 时间线分歧
- 社区投票通过

**混沌线（默认状态）：**
- 任何人在任意节点创建的默认状态
- 未经社区投票接纳

### 2.3 自动升降级机制

| 升级场景 | 条件 | 阈值 |
|---------|------|------|
| 混沌线→支线/平行线 | 社区投票 | 50%+1，30%法定人数 |
| 支线/平行线→主线 | 社区投票 | 67%，40%法定人数 |

| 降级场景 | 条件 |
|---------|------|
| 支线→平行线 | 与主线设定产生冲突 |
| 支线/平行线→混沌线 | 严重违规、长期未更新 |

## 三、社区股份制

### 3.1 股份类型设计

| 股份类型 | 投票权重 | 收益权 | 获得方式 | 转让限制 |
|---------|---------|--------|---------|---------|
| **核心股(A类)** | 1.5x | 1.5x | 主作者初始、核心创作者授予 | 不可转让，仅可继承 |
| **贡献股(B类)** | 1.0x | 1.0x | 付费购买、月票达标自动获得 | 可自由转让 |
| **特邀股(C类)** | 0.5x | 0.5x | 特邀专家、顾问授予 | 不可转让，任期届满回收 |

### 3.2 初始股份分配

**冷启动阶段：**
- 主作者：51%核心股（A类）
- 预留池：49%贡献股（B类）

**成熟阶段目标：**
- 主作者：20-30%核心股
- 创作者社区：60-70%贡献股
- 特邀顾问：5-10%特邀股

### 3.3 持股限制

**防垄断机制：**
- 单一股东持股≤10%（主作者除外）
- 超过10%部分，投票权重降至0.5x
- 关联方合并计算（家庭成员、同IP、一致行动人）

**股份锁定期：**
- 付费购买的股份：180天锁定期
- 锁定期内不得投票，但享受收益权

## 四、股份获取机制

### 4.1 付费购买股份

**定价方式（主作者可选）：**

| 方案 | 说明 | 推荐场景 |
|------|------|---------|
| 固定价格 | 1%股份=固定金额 | 早期IP，价格较低 |
| 阶梯价格 | 阶梯定价，后期更贵 | 成长期IP |
| 拍卖制 | 公开拍卖，价高者得 | 热门IP |

**建议定价参考：**
```
新人IP：1%股份 = 100-500元
成长IP：1%股份 = 500-2000元
成熟IP：1%股份 = 2000-10000元
```

**购买限额：**
- 单次购买：1%-5%
- 个人累计上限：10%
- 超额部分投票权重降至0.5x

### 4.2 创作者自动获取股份

**触发条件（同时满足）：**
1. 作品被社区接纳（非黑名单）
2. 连续3个月月票≥100票
3. 获得至少10名股东推荐

**获股数量计算：**
```
基础股份 = 0.1%
质量加成 = 月票数/10000（最高0.5%）
总股份 = min(基础股份 + 质量加成, 0.6%)
```

**示例：**
| 月票数 | 获得股份 |
|--------|---------|
| 100票 | 0.11% |
| 500票 | 0.15% |
| 1000票 | 0.20% |
| 5000票 | 0.60%（封顶）|

### 4.3 池子分配策略

**主作者配置选项：**

```
=== 池子设置 ===
社区池比例：___ % （建议30%-50%）
├─ 付费购买池：___ %
└─ 创作者自动获取池：___ %

=== 池子分完后 ===
□ 增发稀释模式（社区可无限增长）
□ 主作者出让模式（从主作者份额分）
□ 关门模式（停止接纳，需主作者主动增发）
```

## 五、月票系统

### 5.1 月票本质

月票具有三重属性：
- **投票权**：读者对作品的认可与支持
- **股份触发器**：创作者获取股份的门槛指标
- **价值载体**：可转化为作品收益与创作者收入

### 5.2 月票获取方式

**付费获取：**
```
1元 = 1张月票
10元 = 12张月票（9折）
50元 = 65张月票（87折）
100元 = 140张月票（7折）
500元 = 750张月票（75折）
```

**免费获取：**

| 方式 | 奖励 |
|------|------|
| 每日签到 | 1-3张/天（连续签到递增） |
| 阅读任务 | ≥30分钟/≥10000字 = 1-2张 |
| 社区贡献 | 优质评论/纠错/邀新 = 2-10张 |
| 等级权益 | LV1-LV5每月5-80张 |

### 5.3 月票使用规则

**投票限制：**
- 单作品单读者月票上限：LV1-2(10张) → LV3(30张) → LV4(50张) → LV5(无限)
- 投票冷却：同一作品间隔≥1小时

**有效期：**
- 免费月票：当月有效
- 付费月票：永久有效
- 投票记录：永久保存（计算历史累积）

### 5.4 有效月票（防刷票核心）

**阅读判定标准：**
- 累计阅读时长 ≥ 30分钟
- 阅读字数 ≥ 10000字
- 阅读章节 ≥ 5章

**未达标准的月票计入"初始月票"，不触发股份获取。**

### 5.5 月票阈值（动态调整）

#### 新人作品（发布<3个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 100张有效月票/月 | 0.01% |
| 银牌 | 300张有效月票/月 | 0.05% |
| 金牌 | 1000张有效月票/月 | 0.1% |

#### 成长作品（发布3-12个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 500张有效月票/月 | 0.1% |
| 银牌 | 2000张有效月票/月 | 0.5% |
| 金牌 | 5000张有效月票/月 | 1% |

#### 成熟作品（发布>12个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 2000张有效月票/月 | 0.5% |
| 银牌 | 8000张有效月票/月 | 2% |
| 金牌 | 20000张有效月票/月 | 5% |

**动态调整：** 每月根据平台整体数据调整，单次≤±10%

### 5.6 防刷票机制

**技术层：**
- 实名认证（身份证+人脸）
- 行为分析（IP/设备指纹/时间规律）
- 图灵测试（随机阅读理解题）

**经济层：**
- 刷票惩罚：警告→封禁90天→永久封禁
- 举报奖励：50张月票+没收金额50%

**制度层：**
- 创作者隔离：不能给自己投票
- 资本限制：单投资者月票上限10000张

## 六、投票与治理机制

### 6.1 投票权重计算

```
有效投票数 = Σ(持有股份 × 股份类型权重)
```

**示例：**
- 持有1%核心股(A类) = 1.5票
- 持有1%贡献股(B类) = 1.0票
- 持有1%特邀股(C类) = 0.5票

### 6.2 黄金股机制（主作者一票否决权）

**可否决的场景（需说明理由）：**
1. 修改世界观核心设定（如魔法体系、力量体系）
2. 修改社区核心价值观
3. 解散社区或重大资产处置
4. 修改主作者核心权益条款

**不可否决的场景：**
1. 正常的内容分级投票
2. 创作者贡献奖励分配
3. 日常运营决策
4. 争议仲裁结果（除非涉及核心设定）

**使用限制：**
- 连续3个月内使用不超过2次
- 每次使用必须在社区公示详细理由
- 社区可发起"撤销黄金股"投票（需75%超级多数通过）

### 6.3 投票场景与阈值

| 投票场景 | 通过门槛 | 法定人数 | 冷却期 |
|---------|---------|---------|--------|
| 混沌线→支线 | 50%+1 | 30% | 7天 |
| 混沌线→平行线 | 50%+1 | 30% | 7天 |
| 支线↔平行线 | 50%+1 | 30% | 14天 |
| 支线/平行线→主线 | 67% | 40% | 30天 |
| 主线→支线/平行线 | 50%+1 | 30% | 14天 |
| 黑名单提案 | 75% | 50% | 永久 |
| 解除黑名单 | 67% | 40% | 90天 |
| 修改社区规则 | 67% | 40% | 30天 |
| 修改核心设定 | 75%+主作者同意 | 50% | 60天 |
| 收益分配方案 | 50%+1 | 30% | 90天 |
| 撤销主作者黄金股 | 75% | 60% | 180天 |

### 6.4 投票流程

```
发起提案 → 社区审核(48h) → 投票期(7-14天) → 公示期(3天) → 执行
```

**提案发起资格：**
- 持有≥1%股份的成员
- 社区委员会
- 主作者

**投票期时长：**
- 标准投票：7天
- 重大事项（规则修改、核心设定）：14天
- 紧急事项（社区安全）：3天

### 6.5 累积投票权（保护小股东）

在选举类投票中，小股东可将所有票数集中投给单一候选人：

```
可投票数 = 持有股份 × 候选人数
```

**示例：** 持有0.5%股份的成员，在选举5人委员会时，拥有0.5%×5=2.5%的票数，可全部投给同一人。

## 七、分支机制

### 7.1 分支创建

- **任意节点可分支**：章节、段落、情节点
- **可视化展示**：所有分支可见
- **分支信息**：作者、字数、更新时间、层级、热度

### 7.2 分支树示例

```
主线第1章
├─ 主线第2章
│  ├─ 主线第3章
│  └─ [平行线] 小明的冒险（fork自第2章）
│     └─ [支线] 小明的后续（社区接纳为支线）
└─ [混沌线] 如果反派赢了（fork自第1章）
```

### 7.3 阅读体验

**分支展示：**
- 树状图可视化
- 筛选功能（只看支线、按热度排序）
- 阅读时分支提示

**"接纳"机制说明：**
- 接纳 ≠ 物理合并
- 仅作为"IP认可"标记
- 被接纳内容独立存在，读者可单独阅读

## 八、争议解决机制

### 8.1 三层争议解决体系

#### 第一层：社区调解（7天）

**适用场景：**
- 创作冲突（设定矛盾、角色冲突）
- 轻微违规（非恶意）
- 成员间纠纷

**处理原则：** "包容性多元"优于"统一性"

**解决方案优先级：**
1. **平行宇宙法**: 宣布两者处于不同平行世界
2. **时空线分离**: 归属不同时间线
3. **局部设定法**: 两者均有效，仅在不同地域/阵营生效
4. **投票表决**: 前述方法均失败时，由社区投票决定

#### 第二层：社区仲裁（14天）

**适用场景：**
- 调解失败
- 中度违规（恶意破坏、抄袭）
- 涉及金额较大的争议

**仲裁庭组成：**
- 5名仲裁员（社区委员会3名+随机抽取股东2名）
- 回避制度：利益相关者必须回避

#### 第三层：社区大会

**适用场景：**
- 仲裁结果被申诉
- 重大规则争议
- 撤销主作者黄金股
- 解散社区

### 8.2 抄袭争议处理

**抄袭定义：**
- 连续100字完全相同，或
- 核心创意/情节高度相似，且无法证明独立创作

**处理措施：**
- 首次：警告 + 要求修改
- 二次：支线降级为混沌线
- 三次：加入黑名单 + 收回股份

## 九、防滥用机制

### 9.1 防止"多数人暴政"

**小股东保护机制：**
- 超级多数保护（关键决策需75%）
- 小股东否决权池（持有<1%成员可联合否决，需收集≥10%签名）
- 累积投票权（选举时可集中票数）
- 提案保护（持有≥0.5%即可发起提案）

### 9.2 防止大户垄断

**持股限制：**
- 单一股东持股≤10%（主作者除外）
- 关联方合并计算
- 超额部分投票权重降至0.5x

**提案频率限制：**
- 单一股东：每30天最多发起3个提案
- 联合提案：需至少5个不同股东联署

### 9.3 防止恶意收购

**保护机制：**
- 股份锁定期：180天
- 慢速稀释：单月增持≤1%，超过部分每月释放10%
- 社区优先权：转让时社区享有优先购买权

### 9.4 防止投票操纵

**技术防护：**
- 实名认证（保护隐私，仅后台验证）
- 设备指纹
- 行为分析
- 区块链存证关键决策

## 十、主作者角色

### 10.1 主作者特殊权益

| 权益类型 | 具体内容 |
|---------|---------|
| **投票权** | 1.5x投票权重 |
| **黄金股** | 核心设定一票否决权（受限制） |
| **收益权** | 1.5x收益分成 |
| **永久席位** | 社区委员会永久席位 |
| **提案免审** | 提案跳过审核阶段，直接进入投票 |
| **创作特权** | 可创作任何分级的内容、定义核心世界观 |

### 10.2 主作者责任

**创作义务：**
- 最低创作频率：每月≥1万字
- 保持主线剧情更新
- 维护世界观一致性

**社区义务：**
- 每季度社区大会致辞
- 回应社区重大关切
- 引导社区文化发展

### 10.3 主作者传承与更替

#### 正常传承
**流程：** 主作者提名 → 委员会审核 → 社区大会投票（67%通过） → 交接

**继任者资格：**
- 持股≥5%
- 在社区活跃≥2年
- 获社区委员会认可

#### 非正常更替
**长期不活跃定义：**
- 连续6个月无创作
- 连续3个月不参与社区活动
- 无正当理由

**处理流程：** 暂停特权 → 临时委员会接管 → 90天内选举新主作者

#### 极端罢免
**触发条件：** 严重损害社区利益、严重违反规则、刑事处罚

**流程：** 委员会发起罢免提案 → 75%超级多数通过 → 剥夺核心股

## 十一、退出机制

### 11.1 主动退出

**申请条件：**
- 持股≥6个月
- 无未完成争议/违规处理

**退出流程：** 提交申请 → 委员会审核(30天) → 确定回购价格 → 完成交易

**回购价格计算：**
```
回购价格 = 当前净资产 × 持股比例 × 退出系数

退出系数：
- 持股6-12个月：0.8
- 持股12-24个月：1.0
- 持股24个月以上：1.2
```

### 11.2 被动退出（黑名单）

**触发条件：**
- 加入黑名单
- 严重违反社区规则
- 恶意破坏社区

**回购价格：** 按净资产的50%回购（惩罚性），特殊严重情况：无偿回购

### 11.3 自然退出

**触发条件：** 连续12个月无任何活动

**处理方式：**
- 冻结投票权和收益权
- 保留股份6个月（可申请恢复）
- 6个月后仍未活动，自动回购（按净资产×0.6）

### 11.4 社区解散

**触发条件：** 主动解散（75%投票通过）或 被动解散（连续6个月无法运营）

**解散顺序：**
1. 清算费用
2. 员工/创作者应得报酬
3. 债务偿还
4. 股东按比例分配剩余资产

## 十二、冷启动方案

### 第一阶段：种子期（0-3个月）

**目标：** 建立核心团队，完善世界观

**参与者：**
- 主作者（100%股份）
- 邀请3-5名核心创作者（赠送各2-5%股份）

**治理方式：**
- 主作者决策为主
- 核心创作者提供建议
- 建立初步社区规则

**关键产出：**
- 核心世界观设定
- 主线剧情框架
- 社区治理章程初稿

### 第二阶段：萌芽期（3-6个月）

**目标：** 扩大规模，测试治理机制

**参与者：**
- 开放付费购买股份（限量发行20%）
- 创作者自动获股机制启动

**治理方式：**
- 投票机制正式运行
- 社区委员会成立（主作者+2名核心创作者+2名股东代表）
- 黄金股有限行使

### 第三阶段：成长期（6-12个月）

**目标：** 去中心化，社区自治

**参与者：**
- 开放更多股份（至49%）
- 创作者成为主力

**治理方式：**
- 委员会扩容（11名成员）
- 主作者逐步放权
- 累积投票制全面应用

### 第四阶段：成熟期（12个月+）

**目标：** 完全自治，可持续运营

**治理方式：**
- 主作者持股稳定在20-30%
- 委员会完全自治
- 黄金股仅作最后防线

**关键指标：**
- 月活创作者≥100人
- 月投票参与率≥60%
- 内容分级体系完整

### A. 创作者自动获股计算示例

```
创作者A的作品连续3个月获得1000张月票
基础股份 = 0.1%
质量加成 = 1000/10000 = 0.1%
总股份 = 0.1% + 0.1% = 0.2%
```

### B. 投票权重计算示例

**场景：** 总股份数1000股，投票需达到30%法定人数

**股东分布：**
- 主作者：510股核心股(A类) → 765票
- 股东A：100股贡献股(B类) → 100票
- 股东B：50股贡献股(B类) → 50票
- 其他小股东：340股贡献股(B类) → 340票
- **总票数：1255票**

**法定人数要求：** 1000 × 30% = 300股

### C. 关键术语表

| 术语 | 定义 |
|-----|------|
| **股份** | 社区所有权证明，=投票权=收益权 |
| **核心股(A类)** | 主作者和核心创作者持有，1.5x权重 |
| **贡献股(B类)** | 普通股东持有，1.0x权重 |
| **特邀股(C类)** | 特邀顾问持有，0.5x权重 |
| **黄金股** | 主作者的一票否决权，受限制 |
| **月票** | 读者投票机制，决定创作者获股 |
| **有效月票** | 需真实阅读(≥30分钟+≥10000字) |
| **混沌线** | 未正式接纳的内容，无投票权 |
| **支线** | 能融入主线宇宙的内容 |
| **平行线** | 不能融入主线宇宙但被认可的内容 |
