## 一、核心概念

### 平台定位
类似GitHub的小说创作平台，作者创作主线内容，读者可以在任意节点创建分支，创作平行宇宙故事。通过社区投票机制对内容进行分级，根据不同级别进行收益分成。

### 核心原则

| 原则 | 说明 |
|------|------|
| **三位一体** | 股份 = 投票权 = 收益权，权责对等 |
| **创作者优先** | 作品被接纳+月票达标→自动获股，确保新血液注入 |
| **保护少数** | 多重机制保护小股东权益，防止"多数人暴政" |
| **激励相容** | 所有治理机制激励成员为社区创造价值 |
| **渐进放权** | 随社区成熟，逐步降低主作者特权，向去中心化过渡 |
| **包容性多元** | 创作冲突优先用"平行宇宙法"解决，而非简单否定 |
| **决策执行分离** | 决策层民主制衡，执行层高效负责 |

## 二、层级体系

### 2.1 内容分级

| 层级 | 定义 | 创作者权限 | 投票权 | 收益权 |
|------|------|-----------|--------|--------|
| **主线** | 主作者原创内容，决定世界观发展 | 主作者 | 1股=1票 | 1股=1份收益 |
| **支线** | 能融入主线宇宙的内容 | 自动获股 | 1股=1票 | 1股=1份收益 |
| **平行线** | 不能融入主线宇宙但被认可 | 自动获股 | 1股=1票 | 1股=1份收益 |
| **混沌线** | 未获社区接纳，读者自嗨 | 无 | 无 | 无 |

### 2.2 层级判定标准

**支线（能融入主线宇宙）：**
- 不与主线设定冲突
- 补充主线内容（配角视角、番外等）
- 主线可以引用/提及
- 社区投票通过

**平行线（不能融入主线宇宙但有价值）：**
- 与主线设定有冲突但质量优秀
- 改变主线剧情走向
- 时间线分歧
- 社区投票通过

**混沌线（默认状态）：**
- 任何人在任意节点创建的默认状态
- 未经社区投票接纳

### 2.3 自动升降级机制

| 升级场景 | 条件 | 阈值 |
|---------|------|------|
| 混沌线→支线/平行线 | 社区投票 | 50%+1，30%法定人数 |
| 支线/平行线→主线 | 社区投票 | 67%，40%法定人数 |

| 降级场景 | 条件 |
|---------|------|
| 支线→平行线 | 与主线设定产生冲突 |
| 支线/平行线→混沌线 | 严重违规、长期未更新 |

## 三、社区股份制

### 3.1 主作者配置系统

**主作者创建IP时只需填写3个参数：**

```
┌─────────────────────────────────────────┐
│   创建你的共享小说社区                   │
├─────────────────────────────────────────┤
│                                         │
│   总股数：    [10000] 股                 │
│                                         │
│   你的初始比例： [30] %                 │
│                                         │
│   最低比例：  [15] %                    │
│   （锁定部分，永不稀释）                │
│                                         │
│   ──────────────────────────────────    │
│   📊 系统自动计算：                     │
│   你的股数：1765股                      │
│   社区池：8235股                        │
│   你的最低保障：15%                     │
│                                         │
│   [创建社区]                             │
│                                         │
└─────────────────────────────────────────┘
```

**参数说明：**

| 参数 | 说明 | 限制条件 |
|------|------|---------|
| 总股数 | 社区的总股份数，建议10000或100000 | > 0 |
| 初始比例 | 主作者的初始持股比例 | > 最低比例 |
| 最低比例 | 主作者的最低保障比例，永不稀释 | < 初始比例 |

**系统自动计算公式：**

```
主作者初始股数 = 总股数 × (初始比例 - 最低比例) ÷ (1 - 最低比例)

验证公式：
主作者股权 = 最低比例 + (主作者股数 ÷ 总股数) × (1 - 最低比例) = 初始比例 ✓
```

**计算示例：**

| 配置 | 总股数 | 初始比例 | 最低比例 | 主作者股数 | 社区池 |
|------|--------|---------|---------|-----------|--------|
| 强控制 | 10000股 | 60% | 20% | 5000股 | 5000股 |
| 平衡型 | 10000股 | 30% | 15% | 1765股 | 8235股 |
| 民主型 | 10000股 | 20% | 10% | 1111股 | 8889股 |

**限制条件验证：**

```
✅ 正确配置
初始比例：30%
最低比例：15%
验证：30% > 15% ✓

❌ 错误配置
初始比例：15%
最低比例：15%
错误：初始比例必须大于最低比例
```

### 3.3 股权计算与稀释机制

**主作者股权公式：**

```
主作者股权 = 锁定比例 + (主作者股数 ÷ 当前总股数) × (1 - 锁定比例)
```

**其中：**
- **锁定比例（最低比例）**：按比例计算，永不稀释
- **非锁定部分**：按股数计算，会被稀释

**稀释过程示例：**

| 阶段 | 新人股数 | 当前总股数 | 主作者股数 | 主作者股权计算 | 主作者比例 |
|------|---------|-----------|-----------|--------------|-----------|
| 初始 | 0 | 10000股 | 1765股 | 15% + (1765/10000)×85% | **30%** |
| 第1批 | 1000股 | 11000股 | 1765股 | 15% + (1765/11000)×85% | 28.6% |
| 第10批 | 5000股 | 15000股 | 1765股 | 15% + (1765/15000)×85% | 25.0% |
| 第50批 | 50000股 | 60000股 | 1765股 | 15% + (1765/60000)×85% | 17.5% |
| **极限** | **∞** | **∞** | **1765股** | **15% + 0** | **15%（最低保障）** |

**关键特性：**
- 主作者的**股数永远不变**（1765股）
- 随着总股数增加，主作者比例会下降
- 但**永远不会低于最低比例**（15%）

### 3.4 股权与收益权

**核心原则：股权 = 收益权 = 投票权，三位一体**

| 权益类型 | 说明 |
|---------|------|
| **投票权** | 1股 = 1票，按持股比例行使投票权 |
| **收益权** | 1股 = 1份收益权，按持股比例分配利润 |
| **转让权** | 待补充 |

**说明：** 所有股份权利相同，不再区分A/B/C类。主作者和创作者持有的股份权利完全平等。通过锁定比例机制保护主作者最低持股比例，而非通过不同股份类型。

### 3.5 持股限制

**防垄断机制：**
- 单一股东持股≤10%（主作者除外）
- 关联方合并计算（家庭成员、同IP、一致行动人）

**股份锁定期：**
- 付费购买的股份：180天锁定期
- 锁定期内不得投票，但享受收益权

### 3.6 社区池分完后的处理方式

**主作者创建IP时可选择：**

| 方案 | 说明 | 效果 |
|------|------|------|
| **停止接纳** | 社区池分完后，停止接纳新创作者 | 社区规模固定，保护现有股东权益 |
| **自动增发** | 社区分完后自动增发新池子 | 社区可持续增长，但现有股东会被稀释 |

**两种方案的对比：**

```
【方案A：停止接纳】
初始：总股10000股，主作者1765股，社区池8235股
分完：总股10000股，主作者1765股，创作者8235股
状态：池子空了，停止接纳新人

【方案B：自动增发】
初始：总股10000股，主作者1765股，社区池8235股
分完：自动增发8235股新池子
增发后：总股18235股，主作者1765股（9.7%），第一轮创作者8235股（45.2%），新池子8235股
状态：可继续接纳，主作者比例下降但绝对股数不变
```

**配置界面：**

```
┌─────────────────────────────────────────┐
│   创建你的共享小说社区                   │
├─────────────────────────────────────────┤
│                                         │
│   总股数：    [10000] 股                 │
│                                         │
│   你的初始比例： [30] %                 │
│                                         │
│   最低比例：  [15] %                    │
│                                         │
│   池子分完后：  ○ 停止接纳  ○ 自动增发   │
│                                         │
│   ──────────────────────────────────    │
│   📊 系统自动计算：                     │
│   你的股数：1765股                      │
│   社区池：8235股                        │
│                                         │
│   [创建社区]                             │
│                                         │
└─────────────────────────────────────────┘
```

**方案A：增发稀释模式**
```
主作者股权 = 锁定比例 + (主作者股数 ÷ 新总股数) × (1 - 锁定比例)

继续接纳新创作者，所有人的（非锁定）比例都会被稀释
主作者比例会下降，但永远不会低于锁定比例
```

**方案B：主作者出让模式**
```
主作者从自己的股数中出让一部分给新创作者
主作者股数减少，但锁定比例不变
```

**方案C：关门模式**
```
停止接纳新创作者
除非主作者主动调整配置
```

**示例（增发稀释模式）：**
```
初始：
总股数：10000股
主作者：1765股（30%）
社区池：8235股

社区池分完时（假设8235股全部分完）：
总股数：18235股
主作者：1765股（15% + 1765/18235×85% = 15% + 8.2% = 23.2%）
社区：16470股

继续接纳（新人1000股）：
总股数：19235股
主作者：1765股（15% + 1765/19235×85% = 15% + 7.8% = 22.8%）
```

## 四、股份获取机制

### 4.1 付费购买股份

**定价方式（主作者可选）：**

| 方案 | 说明 | 推荐场景 |
|------|------|---------|
| 固定价格 | 1%股份=固定金额 | 早期IP，价格较低 |
| 阶梯价格 | 阶梯定价，后期更贵 | 成长期IP |

**建议定价参考：**
```
新人IP：1%股份 = 100-500元
成长IP：1%股份 = 500-2000元
成熟IP：1%股份 = 2000-10000元
```

**购买限额：**
- 单次购买：1%-5%
- 个人累计上限：10%

### 4.2 创作者自动获取股份

**触发条件（同时满足）：**
1. 作品被社区接纳（非黑名单）
2. 连续3个月月票≥100票
3. 获得至少10名股东推荐

**获股数量计算：**
```
基础股份 = 0.1%
质量加成 = 月票数/10000（最高0.5%）
总股份 = min(基础股份 + 质量加成, 0.6%)
```

**示例：**
| 月票数 | 获得股份 |
|--------|---------|
| 100票 | 0.11% |
| 500票 | 0.15% |
| 1000票 | 0.20% |
| 5000票 | 0.60%（封顶）|

**股份上限：**
- 单作品累计月票<10000张：0.5%上限
- 单作品累计月票10000-50000张：2%上限
- 单作品累计月票50000-200000张：5%上限
- 单作品累计月票>200000张：10%上限
- **单人总上限：15%**（可多作品累积）

## 五、月票系统

### 5.1 月票本质

月票具有三重属性：
- **投票权**：读者对作品的认可与支持
- **股份触发器**：创作者获取股份的门槛指标
- **价值载体**：可转化为作品收益与创作者收入

### 5.2 月票获取方式

**付费获取：**
```
1元 = 1张月票
10元 = 12张月票（9折）
50元 = 65张月票（87折）
100元 = 150张月票（67折）
500元 = 800张月票（62.5折）
```

**免费获取：**

| 方式 | 奖励 |
|------|------|
| 每日签到 | 1-4张/天（连续签到递增）|
| 阅读任务 | ≥30分钟/≥10000字 = 1-2张 |
| 社区贡献 | 优质评论/纠错/邀新 = 2-10张 |
| 等级权益 | LV1-LV5每月5-100张 |

**免费获取占比：65% | 付费获取占比：35%**

### 5.3 月票使用规则

**投票限制：**
- 单作品单读者月票上限：LV1-2(10张) → LV3(30张) → LV4(50张) → LV5(无限)
- 投票冷却：同一作品间隔≥1小时

**有效期：**
- 免费月票：当月有效
- 付费月票：永久有效
- 投票记录：永久保存（计算历史累积）

### 5.4 有效月票（防刷票核心）

**阅读判定标准（三选二通过）：**
- 累计阅读时长 ≥ 30分钟
- 阅读字数 ≥ 10000字
- 阅读章节 ≥ 5章

**未达标准的月票计入"初始月票"，不触发股份获取。**

### 5.5 月票阈值（动态调整）

#### 新人作品（发布<3个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 100张有效月票/月 | 0.01% |
| 银牌 | 300张有效月票/月 | 0.05% |
| 金牌 | 1000张有效月票/月 | 0.1% |

#### 成长作品（发布3-12个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 500张有效月票/月 | 0.1% |
| 银牌 | 2000张有效月票/月 | 0.5% |
| 金牌 | 5000张有效月票/月 | 1% |

#### 成熟作品（发布>12个月）
| 阈值等级 | 月票要求 | 获取股份比例 |
|---------|---------|-------------|
| 铜牌 | 2000张有效月票/月 | 0.5% |
| 银牌 | 8000张有效月票/月 | 2% |
| 金牌 | 20000张有效月票/月 | 5% |

**动态调整：** 每月根据平台整体数据调整，单次≤±10%

### 5.6 月票收益

**创作者月票收入：**
```
创作者月票收入 = 作品当月月票数 × ¥0.1 × 创作者分成比例

创作者分成比例（按影响力值）：
- 影响力值 < 1000：50%
- 影响力值 1000-5000：60%
- 影响力值 5000-20000：70%
- 影响力值 20000-100000：80%
- 影响力值 > 100000：90%
```

**平台抽成：10%**

### 5.7 防刷票机制

**技术层：**
- 实名认证（身份证+人脸）
- 有效月票验证（≥30分钟+≥10000字+≥5章）
- 行为分析（IP/设备指纹/时间规律）
- 图灵测试（随机阅读理解题）

**经济层：**
- 刷票惩罚：警告→封禁90天→永久封禁
- 举报奖励：50张月票+没收金额50%

**制度层：**
- 创作者隔离：不能给自己投票
- 资本限制：单投资者月票上限10000张

## 六、混合治理机制（最终方案）

经过团队辩论，最终采用**混合治理机制**，结合三层制的制衡优势和委员会制的执行效率。

### 6.1 混合治理架构

```
┌─────────────────────────────────────────────────────┐
│                   共享小说平台                        │
│                   混合治理结构                        │
├─────────────────────────────────────────────────────┤
│                                                       │
│  【决策层】（民主制衡）                               │
│   ├── 股东大会（股份投票权）→ 利益分配              │
│   ├── 荣誉议会（咨询建议+公投发起）→ 规则制定        │
│   └── 全体公投（1人1票）→ 重大事项                  │
│         │                                             │
│         ▼                                             │
│  【执行层】（高效负责）                               │
│   └── 执行委员会（7人）→ 执行决策+日常运营            │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 6.2 权责分配矩阵

| 决策类型 | 决策主体 | 执行主体 | 监督主体 | 触发公投条件 |
|---------|---------|---------|---------|------------|
| **日常运营** | 执行委员会 | 执行委员会 | 股东大会 | 10%股东联名发起 |
| **规则修改** | 股东大会 | 执行委员会 | 荣誉议会 | 5%股东或荣誉议会发起 |
| **核心设定** | 股东大会+主作者同意 | 执行委员会 | 荣誉议会 | 3%股东或荣誉议会发起 |
| **黑名单提案** | 股东大会 | 执行委员会 | 荣誉议会 | 不可发起（防多数人暴政）|
| **收益分配** | 执行委员会提案 | 执行委员会 | 股东大会 | 15%股东联名发起 |
| **主作者更替** | 股东大会 | 执行委员会 | 荣誉议会 | 自动公投 |

### 6.3 股东大会（决策层核心）

**成员：** 所有股份持有者

**投票权重：**
```
有效投票数 = 持有股份数
1股 = 1票
```

**决策范围：**
- 利润分配方案
- 理事会成员选举
- 重大战略方向
- 修改治理规则
- 内容分级（混沌线→支线/平行线）

**投票阈值：**
- 简单决策：50%+1，30%法定人数
- 重要决策：67%，40%法定人数
- 重大决策：75%，50%法定人数

### 6.4 荣誉议会（咨询机构）

**成员：** 21人（象征社区21%预留池）

**产生方式：** 由活跃读者选举（月票≥100张/月）

**权力：**
- 咨询建议权（无约束力）
- 公投发起权（需2/3多数）
- 争议调解权（参与第一层调解）
- **无投票权**（关键决策不参与投票，防止稀释股东权益）

### 6.5 全体公投（最终防线）

**适用场景：**
- 理事会罢免：30%股东联署，≥67%通过
- 核心规则修改：20%社区联署，≥60%通过
- 平台解散/合并：≥75%通过
- 紧急争议裁决：≥51%通过

**投票规则：** 所有人1人1票

### 6.6 执行委员会（执行层）

**组成（7人）：**

| 席位 | 人数 | 产生方式 | 任期 | 职责 |
|------|------|---------|------|------|
| 主作者 | 1人 | 永久席位 | - | 代表核心创作者 |
| 核心创作者代表 | 2人 | 股东大会选举 | 2年 | 代表创作者利益 |
| 股东代表 | 2人 | 股东大会选举（累积投票制）| 2年 | 代表股东利益 |
| 读者代表 | 1人 | 荣誉议会选举 | 2年 | 代表读者利益 |
| 独立专家 | 1人 | 提名+股东大会确认 | 2年 | 提供专业意见 |

**制衡机制：**
- 委员会决策需≥5/7同意
- 股东大会可随时罢免委员（需67%通过）
- 委员会任期1年，可连任1次

**职责：**
- 执行股东大会和荣誉议会决议
- 管理平台日常运营
- 提交议案和年度报告
- 处理紧急事项
- 组织投票流程

### 6.7 黄金股机制（主作者一票否决权）

**可否决的场景（需说明理由）：**
1. 修改世界观核心设定（如魔法体系、力量体系）
2. 修改社区核心价值观
3. 解散社区或重大资产处置
4. 修改主作者核心权益条款

**不可否决的场景：**
1. 正常的内容分级投票
2. 创作者贡献奖励分配
3. 日常运营决策
4. 争议仲裁结果（除非涉及核心设定）

**使用限制：**
- 连续3个月内使用不超过2次
- 每次使用必须在社区公示详细理由
- 社区可发起"撤销黄金股"投票（需75%超级多数通过）

### 6.8 投票流程

**标准投票流程：**
```
发起提案 → 理事会审核(48h) → 投票期(7-14天) → 公示期(3天) → 执行
```

**发起资格：**
- 持有≥1%股份的成员
- 执行委员会
- 主作者

**投票期时长：**
- 标准投票：7天
- 重大事项（规则修改、核心设定）：14天
- 紧急事项（社区安全）：3天

### 6.9 累积投票权（保护小股东）

在选举类投票中，小股东可将所有票数集中投给单一候选人：

```
可投票数 = 持有股份 × 候选人数
```

**示例：** 持有0.5%股份的成员，在选举5人委员会时，拥有0.5%×5=2.5%的票数，可全部投给同一人。

## 八、争议解决机制

### 8.1 三层争议解决体系

#### 第一层：社区调解（7天）

**适用场景：**
- 创作冲突（设定矛盾、角色冲突）
- 轻微违规（非恶意）
- 成员间纠纷

**处理原则：** "包容性多元"优于"统一性"

**解决方案优先级：**
1. **平行宇宙法**: 宣布两者处于不同平行世界
2. **时空线分离**: 归属不同时间线
3. **局部设定法**: 两者均有效，仅在不同地域/阵营生效
4. **投票表决**: 前述方法均失败时，由社区投票决定

#### 第二层：社区仲裁（14天）

**适用场景：**
- 调解失败
- 中度违规（恶意破坏、抄袭）
- 涉及金额较大的争议

**仲裁庭组成：**
- 5名仲裁员（执行委员会3名+随机抽取股东2名）
- 回避制度：利益相关者必须回避

#### 第三层：社区大会

**适用场景：**
- 仲裁结果被申诉
- 重大规则争议
- 撤销主作者黄金股
- 解散社区

### 8.2 抄袭争议处理

**抄袭定义：**
- 连续100字完全相同，或
- 核心创意/情节高度相似，且无法证明独立创作

**处理措施：**
- 首次：警告 + 要求修改
- 二次：支线降级为混沌线
- 三次：加入黑名单 + 收回股份

## 九、防滥用机制

### 9.1 防止"多数人暴政"

**小股东保护机制：**
- 超级多数保护（关键决策需75%）
- 小股东否决权池（持有<1%成员可联合否决，需收集≥10%签名）
- 累积投票权（选举时可集中票数）
- 提案保护（持有≥0.5%即可发起提案）

### 9.2 防止大户垄断

**持股限制：**
- 单一股东持股≤10%（主作者除外）
- 关联方合并计算

**提案频率限制：**
- 单一股东：每30天最多发起3个提案
- 联合提案：需至少5个不同股东联署

### 9.3 防止恶意收购

**保护机制：**
- 股份锁定期：180天
- 慢速稀释：单月增持≤1%，超过部分每月释放10%
- 社区优先权：转让时社区享有优先购买权（待补充）

### 9.4 防止投票操纵

**技术防护：**
- 实名认证（保护隐私，仅后台验证）
- 设备指纹
- 行为分析
- 区块链存证关键决策

## 十、主作者角色

### 10.1 主作者特殊权益

| 权益类型 | 具体内容 |
|---------|---------|
| **投票权** | 1股 = 1票（与其他股东平等） |
| **黄金股** | 核心设定一票否决权（受限制） |
| **收益权** | 1股 = 1份收益（与其他股东平等） |
| **永久席位** | 执行委员会永久席位 |
| **提案免审** | 提案跳过审核阶段，直接进入投票 |
| **创作特权** | 可创作任何分级的内容、定义核心世界观 |

### 10.2 主作者责任

**创作义务：**
- 最低创作频率：每月≥1万字
- 保持主线剧情更新
- 维护世界观一致性

**社区义务：**
- 每季度社区大会致辞
- 回应社区重大关切
- 引导社区文化发展

### 10.3 主作者传承与更替

#### 正常传承
**流程：** 主作者提名 → 委员会审核 → 社区大会投票（67%通过） → 交接

**继任者资格：**
- 持股≥5%
- 在社区活跃≥2年
- 获社区委员会认可

#### 非正常更替
**长期不活跃定义：**
- 连续6个月无创作
- 连续3个月不参与社区活动
- 无正当理由

**处理流程：** 暂停特权 → 临时委员会接管 → 90天内选举新主作者

#### 极端罢免
**触发条件：** 严重损害社区利益、严重违反规则、刑事处罚

**流程：** 委员会发起罢免提案 → 75%超级多数通过 → 剥夺股份

## 十一、退出机制

### 11.1 主动退出

**申请条件：**
- 持股≥6个月
- 无未完成争议/违规处理

**退出流程：** 提交申请 → 委员会审核(30天) → 确定回购价格 → 完成交易

**回购价格计算：**
```
回购价格 = 当前净资产 × 持股比例 × 退出系数

退出系数：
- 持股6-12个月：0.8
- 持股12-24个月：1.0
- 持股24个月以上：1.2
```

### 11.2 被动退出（黑名单）

**触发条件：**
- 加入黑名单
- 严重违反社区规则
- 恶意破坏社区

**回购价格：** 按净资产的50%回购（惩罚性），特殊严重情况：无偿回购

### 11.3 自然退出

**触发条件：** 连续12个月无任何活动

**处理方式：**
- 冻结投票权和收益权
- 保留股份6个月（可申请恢复）
- 6个月后仍未活动，自动回购（按净资产×0.6）

### 11.4 社区解散

**触发条件：** 主动解散（75%投票通过）或 被动解散（连续6个月无法运营）

**解散顺序：**
1. 清算费用
2. 员工/创作者应得报酬
3. 债务偿还
4. 股东按比例分配剩余资产

## 附录：股本结构修正说明

### 修正背景

在最终版v3.0发布后，主作者提出了更精确的股本结构需求，确保主作者在稀释过程中有最低保障。

### 核心修正

**原设计问题：**
- 股份按固定数量分配，容易导致理解混乱
- 稀释机制不够清晰

**修正后的设计：**
- 引入**锁定比例**（最低保障）概念
- 主作者股权 = 锁定比例 + (主作者股数 ÷ 当前总股数) × (1 - 锁定比例)
- 锁定部分按比例计算，永不稀释
- 非锁定部分按股数计算，可被稀释

### 主作者配置简化为主作者只需填写3个参数：
1. 总股数
2. 初始比例
3. 最低比例（锁定比例）

系统自动计算主作者初始股数，无需手动配置。
